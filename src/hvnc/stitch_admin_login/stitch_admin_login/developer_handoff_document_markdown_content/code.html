TECHNICAL PROJECT SPECIFICATION: HVNC REMOTE ACCESS PLATFORM

1. Project Overview

A high-performance remote access platform designed for managed Chrome browser sessions. The system consists of an Admin Dashboard for infrastructure management and a User Portal for authenticated remote access.

2. Design System (Charcoal & Orange)

Primary Background: linear-gradient(135deg, #333333, #F6921E)

Surface/Card: #2B2B2B (Semi-transparent, 12px backdrop-blur)

Primary Action: #333333 (White text)

Secondary Action: #F6921E (White text)

3. Screen Inventory & Functional Requirements

  • Admin Dashboard: Real-time stats, live device grid, activity feed.
  • Device Management: IP tracking, Hubstaff timer controls, code generation.
  • User Management: Profiles, statistics, device assignment mapping.
  • Shift Scheduling: Calendar view for user/device shift mapping.
  • User Portal: 8-character code entry, connection sequence, hidden Chrome session.

4. Technical Architecture

Frontend: React/Next.js responsive desktop-first baseline (1440px).

Backend: Node.js/Python for JWT Auth, WebSocket heartbeats, and E2EE streaming.

This document serves as the primary handoff for development teams to build the HVNC Remote Access platform based on the generated designs.
